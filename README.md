Hop is a **delay-tolerant mesh**: end-to-end encrypted datagrams that hop device to device, over BLE,
Wi-Fi, and the internet, until they reach the person you meant. Held, never dropped.

`libhop` is `hop-core` compiled to a **flat C ABI**. One `hop_*` function surface (`hop.h`, generated by
cbindgen), one poll-model contract, and every SDK, Swift, Kotlin, Go, Python, Ruby, Crystal, C, C++, and
an ESP32, binds the same bytes. Write a bearer that moves opaque frames and a loop that drains and polls,
and you have a real Hop node. It ships as prebuilt static and dynamic libraries per platform, plus the
`hop` crate on crates.io for Rust callers and header regeneration.

## Install

Grab a prebuilt binary for your platform (the static `libhop.a` for mobile/embedded, the dynamic
`libhop.{dylib,so,dll}` for host FFI) and the header, or build both from the crate:

```sh
cargo build -p hop --release      # produces target/release/libhop.{a,dylib,so}
./regen-header.sh                 # regenerates include/hop.h with the pinned cbindgen
```

Point your build at `include/hop.h` and link `-lhop`. The `minimal` feature drops SQLite and the
ergonomic layer for constrained targets (`cargo build -p hop --no-default-features --features minimal`).

## Bind it from C

Two nodes wired by a loopback bearer, then A sends B an untraceable message, all through the header:

```c
#include "hop.h"

// Feed one node's drained bytes straight into the peer.
static void forward(void *ctx, uint64_t link, const uint8_t *bytes, size_t len) {
    hop_bytes_received((const HopNode *)ctx, 1, bytes, len);
}
static bool on_message(void *ctx, const uint8_t *inbox_id, const uint8_t *from,
                       const char *ct, const uint8_t *body, size_t len,
                       uint8_t hops, uint64_t created) {
    printf("got %.*s\n", (int)len, body);
    *(int *)ctx = 1;
    return true; // local persistence succeeded, so core may release its ACK
}

int main(void) {
    const HopNode *a = hop_node_new();
    const HopNode *b = hop_node_new();

    uint64_t now = 1700000000000ULL;         // real clock so prekey adverts aren't judged expired
    hop_node_tick(a, now); hop_node_tick(b, now);
    hop_publish_prekey(a); hop_publish_prekey(b);

    hop_link_up(a, 1, HopLinkRole_Dialer);   // A dialed, B accepted; same link id each side
    hop_link_up(b, 1, HopLinkRole_Acceptor);

    uint8_t b_addr[32], msg_id[32]; int got = 0;
    hop_node_address(b, b_addr);
    const char *text = "meet at the ridge";
    hop_send_message(a, b_addr, "text/plain", (const uint8_t *)text, strlen(text), true, msg_id);

    for (int i = 0; i < 400 && !got; i++) {
        hop_drain_outgoing(a, forward, (void *)b);
        hop_drain_outgoing(b, forward, (void *)a);
        hop_poll_inbox(b, on_message, &got);
        now += 100; hop_node_tick(a, now); hop_node_tick(b, now);
    }
    hop_node_free(a); hop_node_free(b);
}
```

`examples/smoke.c` is this end to end (message + delivery ACK + base58 round-trip + a full `hops://`
service call); `./examples/smoke.sh` builds and runs it against the freshly generated header.

## The contract

- **Poll-model, never callbacks-from-nowhere.** The core never pushes asynchronously. You `hop_node_tick`
  the clock, `hop_drain_outgoing` to the bearer, and `hop_poll_inbox` / `hop_poll_service_*` to read.
- **Inbox acceptance is explicit.** An inbox callback returns true only after it has synchronously
  persisted the stable inbox id and message. False retains the item and withholds its ACK/vaccine.
- **The bearer is four calls.** `hop_link_up` / `hop_bytes_received` / `hop_drain_outgoing` /
  `hop_link_down` move opaque bytes. The core owns every byte of crypto.
- **Versioned and asserted.** `HOP_ABI_VERSION` is baked into the header; every wrapper checks
  `hop_abi_version()` at load, so a wrapper built against a newer header fails loudly instead of drifting.
- **Panics are dropped operations.** A hostile or malformed input degrades to a NULL/false return, never
  an unwind across the FFI boundary.
- **Storage is yours.** `hop_node_open` persists to SQLite; `hop_node_open_keyed` encrypts at rest with a
  key you hold; `hop_node_new` runs ephemeral.

## Status

Prototype. The full node surface, addressing, the `hops://` request/response path, and encrypted-at-rest
storage are built and covered by the C smoke plus every language wrapper's suite. `HOP_ABI_VERSION` is
currently `3`; the header is regenerated in the same change as any ABI edit and guarded in CI.

## The Hop family

`libhop` is what the non-Rust SDKs bind. Same surface, your language:
[node](https://github.com/hopmesh/hop-sdk-node) ·
[python](https://github.com/hopmesh/hop-sdk-python) ·
[go](https://github.com/hopmesh/hop-sdk-go) ·
[ruby](https://github.com/hopmesh/hop-sdk-ruby) ·
[crystal](https://github.com/hopmesh/hop-sdk-crystal) ·
[elixir](https://github.com/hopmesh/hop-sdk-elixir) ·
[apple](https://github.com/hopmesh/hop-sdk-apple) ·
[android](https://github.com/hopmesh/hop-sdk-android).
The protocol core is [hop-core](https://github.com/hopmesh/hop-core); the browser build is
[hop-wasm](https://github.com/hopmesh/hop-wasm).
